本页按项目前端配置入口整理，重点说明每个模块负责什么、关键字段怎么填、路径如何保持一致，以及 Emby、CloudDrive2、115、STRM、订阅、整理和插件之间的协作关系。

功能总览

01

115 网盘

提供文件、分享、离线、秒传、签到和资源转存能力。

02

05

准备工作

安装 CloudDrive2 并挂载 115

登录 115 账号后添加挂载点。源目录直接选择 115 根目录，挂载点选择 /CloudNAS，高级设置里的权限位置填写 0777。

  • 本地目录映射：宿主机目录需要先映射到 CD2 容器的 /CloudNAS，dian115 再挂载同一个宿主机目录。
  • API Key：在 CD2 里申请 API Key，建议直接申请根目录权限，避免后续目录树读取不完整。
  • 云盘名称：记住 CD2 中显示的云盘名称，例如 115open，后续 STRM 和 Navidrome 都会用到。

安装 Emby 和 Madby 插件

Emby 需要映射媒体目录和 /CloudNAS。媒体目录用于扫描 STRM 或整理后的文件，/CloudNAS 用于 Madby 和路径匹配。

  • 媒体路径：Emby、dian115、整理规则里的路径要保持容器内视角一致，例如都使用 /media。
  • 转码与硬件加速：按你的 Emby 镜像配置设备映射、驱动变量和转码目录；dian115 不接管 Emby 转码本身。
  • Emby API Key：在 Emby 后台申请，用于代理配置、媒体库读取、媒体画像、封面生成和部分插件。
  • Madby：安装到 Emby 插件目录后，在插件配置中填写 dian115 地址和 OpenAPI Key。

准备 dian115 运行目录

推荐使用 Docker Compose。配置目录、媒体目录和 CD2 挂载目录要固定下来，后续不要频繁改容器路径。

  • /config：保存配置文件、数据库、插件运行数据和任务状态。
  • /media：媒体库目录，需与 Emby 媒体库挂载路径一致。
  • /CloudNAS：CloudDrive2 挂载目录，必须使用 :shared 让挂载传播生效。

项目组织地址：madbrolab。

部署启动

Docker Compose

创建目录并编写 docker-compose.yml：

mkdir dian115 && cd dian115
services:
  dian115:
    image: madbrolab/dian115
    container_name: dian115
    restart: unless-stopped
    network_mode: host
    volumes:
      - ./config:/config
      - ./media:/media
      - ./CloudNAS:/CloudNAS:shared

启动服务：

docker compose up -d

首次访问 http://<你的IP>:8095 需要设置管理密码。

网络模式与端口

network_mode: host 是推荐模式，容器直接使用宿主机网络，不需要额外映射端口。

端口用途
8095Web 管理界面。
8098Emby 反向代理默认端口，可在媒体库配置中修改。
4534Navidrome 反向代理默认端口，可在 Navidrome 页面修改。

如果使用 bridge 网络，请自行映射 8095 和所有代理端口。

路径与权限

路径问题是大多数播放失败和同步失败的根源。先把路径统一，再配置 STRM、Emby 代理和整理规则。
路径或字段 应该怎么理解 常见示例
/CloudNAS CD2 的容器挂载目录，也是 dian115 和 Emby 需要能看到的云盘根入口。 ./CloudNAS:/CloudNAS:shared
CD2 挂载前缀 dian115 系统设置里的完整云盘挂载路径，用来把本地路径转换为 115 路径。 /CloudNAS/CloudDrive/open115
STRM 云盘名称 CD2 中实际显示的云盘名称，不是自定义备注。 115open
/media Emby 和 dian115 都能访问的本地媒体库目录，STRM 输出和整理目标通常放这里。 ./media:/media
权限 0777 CD2 挂载点权限位置填写 0777，避免容器间读写目录树或生成 STRM 时权限不足。 CD2 挂载点高级设置

正确顺序

先让 CD2 挂载正常，再启动 dian115；先保存 CD2 地址和 Token，再添加 STRM 规则；先确认 Emby 能扫描媒体库，再让客户端走代理端口。

路径视角

不要混用宿主机路径和容器路径。前端选择器、STRM 输出、Emby 媒体库、整理规则都应使用容器内可见路径。

修改路径

路径改动后要重新检查 STRM 规则、Navidrome 配置、Emby 媒体画像路径转换和 Madby 数据目录。

初始配置

115 账号

入口：115 账号

用于提供 115 接口凭证。账号可通过扫码登录或 Cookie 登录添加，支持多个账号备用。

设备类型
扫码登录时选择登录设备。不同设备类型会影响 Cookie 来源和可用性。
自动签到
用于每日签到任务，可配合通知插件推送结果。
多账号备用
主账号失效或触发限制时，系统可尝试切换备用账号。
QPS 限制
限制 115 接口请求频率。遇到限流、风控或接口失败增多时调低。
安全码
同播复制、清理、部分文件操作会用到。没有这些需求可先不填。
分享 / 离线目录
控制分享转存和磁力离线任务的默认落点，建议建独立目录便于整理。

CloudDrive2

入口：系统设置 → 基础设置

用于连接 CD2 API，并告诉 dian115 115 云盘在本地挂载树中的真实位置。

CD2 地址
同机 host 网络常用 http://localhost:19798。跨机器部署时填写实际内网地址。
API Token
填写 CD2 里申请的 Token。权限不足会导致路径浏览、目录树和文件操作异常。
115 挂载前缀
填写完整前缀，例如 /CloudNAS/CloudDrive/open115，用于本地路径与 115 路径转换。

安全与 OpenAPI

入口：系统设置 → 安全与 API

用于外部系统认证，包括 Madby 插件、Webhook、子服务和第三方调用。

管理密码
Web 管理界面的登录密码。首次访问后应立即设置。
二次验证
需要更高安全性时启用，适合公网反代管理端的场景。
OpenAPI Key
生成后妥善保存。Madby、Webhook 和外部接口请求都可以使用它认证。

代理与过盾

入口：系统设置 → 基础设置

用于解决外部网络访问和 Cloudflare 防护站点访问问题。

HTTP 代理
填写代理主机、端口、用户名和密码。TMDB、Telegram、PT 站点等外部请求可按后端逻辑使用。
FlareSolverr
填写服务地址，例如 http://127.0.0.1:8191。仅访问有防护的站点时需要。

AI 与词表

入口：系统设置 → 高级设置 / 词表设置

主要服务于媒体整理识别，减少文件名不规范时的误判。

AI 模型
填写协议、Base URL、API Key、模型名、上下文限制、温度、Top P、工具调用策略和超时。
识别词
补充常见别名、制作组、特殊命名，帮助识别器更稳定地提取标题和版本字段。
屏蔽词
整理时忽略无意义或干扰识别的词，例如站点宣传、压制标记、广告字段。

系统任务

入口：系统设置 → 系统设置

控制后台任务的并发、日志、队列和 Emby ItemID 深度校验。

日志保留
设置日志保留天数，便于排查播放、同步、订阅和整理问题。
Worker 并发
控制后台任务并发。机器弱或 115 接口不稳定时不要盲目调高。
队列大小
限制待处理任务数量，避免大量同步或整理任务挤满资源。

STRM 规则

云盘名称不是随便写。这里必须填写 CD2 里的实际云盘名称，例如 115open。它和系统设置里的 CD2 挂载前缀不是同一个字段。
配置项 说明 建议
规则名称 用于识别这条同步规则。 按媒体库命名，例如“电影 115open”“剧集 115open”。
云盘名称 CD2 中实际云盘名，用于源路径拼接和目录树转换。 例如 115open，不要填 /CloudNAS。
源路径 115/CD2 中需要生成 STRM 的目录。 使用路径选择器选择，避免手写路径漏掉云盘层级。
输出路径 STRM 文件生成到本地哪里，通常是 Emby 扫描的媒体库目录。 例如 /media/Movies、/media/TV。
链接模式 path 按路径生成；link 使用服务地址生成可访问链接。 常规 Emby 本地扫描优先用路径模式。
服务地址 链接模式下写入 STRM 的服务地址。 需要客户端能访问该地址。
同步模式 手动、定时、实时。定时使用 Cron；实时监听变化。 第一次先手动同步，确认路径正确后再启用自动。
文件扩展名 控制哪些媒体文件生成 STRM。 按库类型保留常见视频或音频扩展名，减少无效文件。
排除目录 同步时跳过指定目录。 排除花絮、样片、回收站、下载临时目录。
元数据扩展名 允许同步海报、字幕、NFO 等旁路文件。 需要 Emby 本地识别元数据时开启。
智能清理 源文件删除后自动清理对应 STRM、元数据和空目录。 确认规则稳定后再打开，避免误清理。
同步字幕 把外挂字幕同步到 STRM 输出目录。 外挂字幕依赖本地文件时建议开启。

推荐配置流程

  • 先在系统设置中保存 CD2 地址、API Token 和 115 挂载前缀。
  • 进入 STRM 规则，填写实际云盘名称并选择源路径。
  • 输出路径选择 Emby 已映射的媒体目录。
  • 先执行手动同步，确认输出目录下 STRM 文件和目录结构正确。
  • 让 Emby 扫描媒体库，客户端通过 dian115 代理端口播放验证。
  • 确认稳定后再启用定时、实时和智能清理。

同步模式选择

  • 手动：适合首次配置、迁移、排错和少量目录。
  • 定时：适合资源变化不频繁的库，通过 Cron 控制增量和全量同步。
  • 实时：适合频繁变化的目录，但依赖挂载和事件监听稳定性。
  • 全量同步：用于修复目录树、漏同步或批量路径调整后的重新校准。

Emby 与 Madby

Emby 代理配置

入口：我的媒体 → 媒体库配置。添加代理后，客户端连接 dian115 的代理端口，而不是直接连接 Emby 原端口。

代理名称
用于区分实例，例如“主 Emby”。当前前端提示仅支持单实例。
Emby 地址
填写后端 Emby 地址，例如 http://192.168.1.100:8096 或同机 http://localhost:8096。
API Key
从 Emby 后台申请。用于读取媒体库、同步 ItemID 和代理匹配。
代理端口
默认 8098。客户端服务器地址填写 http://<你的IP>:8098。
展示内容
选择允许展示的 Emby 媒体库。不选择时按默认逻辑展示。

代理开关含义

  • 启用代理：打开后代理端口才可用，修改配置会热更新。
  • 本地代理：不做 302 跳转，直接代理视频数据。适合直链不可用但希望统一入口的场景。
  • 回退本地：302 跳转失败时改走本地代理，提高播放成功率。
  • 同播复制：检测多设备播放同一视频时执行复制逻辑，需要 115 安全码配合。
  • 同步 Emby ItemID：用于建立 Emby 条目和本地路径/115 文件之间的匹配关系。
Madby 配置页 关键配置 作用
连接 启用插件、dian115 地址、OpenAPI Key、最大并行数、数据目录。 让 Emby 插件能调用 dian115，并保存插件运行数据。
FFP Patch ffprobe、入库自动队列、跳过已有媒体流、严格校验文件大小、Inline 超时。 改善 STRM 媒体流注入和 Emby 入库效率。
扫描增强 STRM 探测输入重定向、HTTP 输入加引号、清理内嵌描述、外置 ffprobe。 减少 ffprobe 对 STRM 或 HTTP 地址的误解析。
快照 媒体信息 JSON 持久化、注入后自动保存、FFP 前恢复、文件变更校验、快照根目录。 缓存媒体流结果，降低重复扫描成本。
媒体库 外挂字幕扫描、缺图刷新、清理失效字幕、语言标准化、多版本合并、章节清理、人物清理。 补齐 Emby 扫描后的非 FFP 增强能力。
字幕 SubHD 地址、语言偏好、候选数量、最大下载大小。 为媒体自动搜索和下载外挂字幕。
体验 界面增强、隐藏无图人物、只显示演员、禁用合集自动创建、拼音排序、深度删除、HTTP 代理。 改善 Emby 管理体验和特定维护动作。
维护 / 内存 元数据标题清理、标题噪声词、定时内存清理、大对象堆整理、归还内存。 处理长期运行下的元数据和内存维护。
Madby 中的 dian115 地址填写 Web 管理地址即可，末尾不需要斜杠；OpenAPI Key 使用 dian115 系统设置中生成的 Key。

订阅系统

订阅入口

订阅页基于 TMDB 发现和搜索：流行趋势、正在热映、热门电影、热门剧集、高分电影等列表都可以发起订阅。

  • 媒体类型：电影或剧集。剧集订阅可按季处理。
  • 订阅状态：会区分 PT 订阅、聚合订阅以及历史冲突状态。
  • 默认行为：系统设置中可配置订阅按钮默认走 PT、聚合或每次询问。
  • 癫影资源：订阅详情中可查看癫影资源，用于手动或聚合流程辅助。

PT 站点配置

入口：PT 站点。页面分为站点配置、搜索与下载设置、下载任务。

  • 站点认证：按站点要求填写 Cookie、API Key、Authorization、Token、User-Agent。
  • 搜索 / RSS：分别控制站点是否参与搜索、RSS 订阅，以及 RSS 地址和超时时间。
  • 优先级：多个站点都命中时，优先级影响选种排序。
  • 代理：单站点可按需要走 HTTP 代理。

下载器配置

支持 qBittorrent 和 Transmission。填写 WebUI 地址、用户名、密码、默认保存路径，并设置默认下载器。

  • 保存路径建议与媒体整理源路径对应，便于下载完成后自动整理。
  • 下载器 WebUI 必须能从 dian115 容器访问。
  • PT 刷流也会复用下载器配置。

PT 策略

策略决定什么时候搜、搜哪些站、选哪个种、是否下载。

  • 自动搜索：按间隔检查订阅，支持多名称搜索。
  • RSS 匹配：从站点 RSS 中快速匹配新资源。
  • 节流参数：站点间休眠、最大翻页、页间休眠用于降低请求压力。
  • 过滤条件：包含词、排除词、只要免费、最少做种、最大体积。
  • 排序：按质量规则、体积、做种、发布时间和站点优先级综合处理。
质量规则 含义 示例
& 同一层级内必须同时满足，表示 AND。 4K&HEVC&HDR
> 不同层级的优先顺序，越靠前越优先。 4K&DV>4K&HEVC>1080P&HEVC
! 排除某个标签。 1080P&!AV1

聚合订阅

聚合订阅可从 TG 资源频道和癫影资源中寻找匹配资源。系统设置里可设置默认来源、是否启用订阅、质量规则组和癫影检查间隔。

  • TG 严格模式：开启后必须命中质量规则才转存；关闭后命中 TMDB ID 即可转存。
  • 质量规则组：可按电影和剧集分别选择预设。
  • 检查间隔：新建订阅会立即检查一次，之后按间隔检查活跃意图。

订阅落地建议

PT 下载完成后通常进入下载目录，再由媒体整理移动或硬链接到 /media。聚合或分享转存的资源则先进入 115 指定目录，再由 CD2、STRM 和 Emby 扫描串起来。

媒体整理

文件管理

入口：文件管理

用于浏览本地媒体目录，支持打开文件夹、预览、下载、识别、整理、重命名、移动、复制、复制路径和删除。

  • “识别”会展示标题、媒体类型、季集、年份和 TMDB 匹配信息。
  • “整理”会按整理规则匹配目标路径并执行移动或硬链接。
  • 适合处理少量异常文件，批量任务建议使用整理规则。

全局设置

入口：媒体整理 → 全局设置

控制命名模板、整理行为、洗版评分和字段命名映射。

  • 电影 / 剧集模板：使用 {变量} 占位符，斜杠表示目录层级。
  • 多版目录模板：用于多版本洗版或保留多版本时的目录结构。
  • 动漫独立整理：开启后动漫可以使用独立分类规则。
  • 并发整理数：默认保守即可，机械盘或弱机器不建议调太高。
  • 字段映射：把识别出的值统一成自定义写法，例如 2160P 统一成 4K。

整理规则

入口：媒体整理 → 整理规则

规则决定从哪里整理到哪里，以及使用什么策略处理重复和多版本。

  • 源路径 / 目标路径：选择容器内可见路径，不要填写宿主机路径。
  • 整理方式：移动或硬链接。硬链接要求源和目标在同一文件系统。
  • 一级类型目录：按电影、剧集、动漫建立顶层目录。
  • 二级分类：结合分类配置输出到华语、欧美、动画等目录。
  • 同步字幕：移动或硬链接媒体时同步外挂字幕。
  • 自动监控：监听源路径变化，自动触发整理。

重复与洗版

规则内配置

重复文件策略用于处理同一媒体已有目标文件的情况。

  • 跳过：目标存在时不处理，最稳妥。
  • 保留全部：全部落地，适合人工后续筛选。
  • 大小比较：按文件大小保留更合适的版本。
  • 评分洗版：按分辨率、片源、编码、HDR、音频、字幕等评分保留。
  • 多版本洗版：按槽位规则保留多个版本，例如 4K DV、4K HDR、1080P。

前置过滤与 AI

规则内配置

前置过滤用于在整理前排除不想要的版本，AI 用于文件名不规范时增强识别。

  • 前置过滤：按字段和值排除资源，可选择被排除时删除源文件。
  • 正则无标题时调用 AI：文件名无法提取标题才调用 AI。
  • TMDB 未命中时调用 AI：正则结果无法匹配 TMDB 时辅助重识别。
  • 始终调用 AI：识别更积极，但成本和耗时更高。
  • FFprobe 解析：读取媒体流信息辅助命名、评分和洗版。

分类、测试与历史

媒体整理页面

分类配置、识别测试和整理历史用于把规则调准。

  • 分类配置：按媒体类型、TMDB 类型、原始语言、制片国家输出到不同分类目录。
  • 识别测试：输入文件名快速验证正则、TMDB 和 AI 结果。
  • 整理历史：按状态、模式、关键词筛选，支持批量删除、批量改名、清理目标文件。

辅助工具

Navidrome 代理

入口：我的媒体 → NAVIDROME

用于把 115 音乐资源通过 Navidrome/Subsonic 协议稳定播放。

  • 填写配置名称、后端地址、用户名、密码和代理端口，默认 4534。
  • 选择 CD2 源路径、本地路径和同步模式。
  • 云盘名称同样填写 CD2 里的实际名称，例如 115open。
  • 支持构建目录树、增量同步、全量同步、302 回退本地和智能清理。

卫星服务

入口：卫星服务

用于发布 STRM 规则目录树，并给子服务客户端分配 API Key。

  • 选择 STRM 规则发布文件树，子服务可通过 API 获取对应规则文件信息。
  • 添加子服务时填写名称、允许规则、过期天数和备注。
  • API Key 只在创建时显示一次，需要妥善保存。
  • 请求日志可查看客户端、类型、详情、状态和 IP。

缓存管理

入口：缓存管理

集中维护 TMDB、115 分享、TG 监控分享和离线链接缓存。

  • TMDB 缓存减少重复请求并提高订阅和整理页面加载速度。
  • 115 分享缓存用于避免重复解析同一分享链接。
  • TG 监控分享缓存用于频道监控去重和状态追踪。
  • 离线链接缓存用于磁力、ED2K 等离线任务去重和复用。

ED2K / 离线任务

入口：相关任务页

用于管理离线链接相关任务。配置前先确认 115 账号可用，并设置好离线下载目录。

  • 建议把离线目录和整理源目录分开，避免未完成文件被整理。
  • 失败任务优先检查账号状态、链接有效性和 115 离线配额。

运维排查

上线前检查

  • CD2 能在 /CloudNAS 看到 115 根目录。
  • dian115 系统设置里的 CD2 地址、API Token、挂载前缀保存并测试成功。
  • Emby、dian115、CD2 的媒体路径一致，Emby API Key 可用。
  • STRM 规则里的云盘名称填的是 CD2 实际云盘名称。
  • 手动同步 STRM 后，Emby 可以扫描到媒体条目。
  • 客户端连接 dian115 代理端口后能播放网盘内容。
  • Madby 插件能连接 dian115，OpenAPI Key 校验正常。

常见问题

  • 路径浏览为空：检查 CD2 地址、Token、根目录权限和 /CloudNAS 挂载传播。
  • STRM 生成失败：检查输出目录权限、云盘名称、源路径和排除规则。
  • Emby 能扫但不能播：检查客户端是否连接代理端口、ItemID 是否同步、302 是否失败、回退本地是否开启。
  • 直链不稳定：检查 115 账号状态、QPS、Cookie 过期、备用账号和网络代理。
  • 订阅不下载：检查站点认证、下载器连接、质量规则、包含/排除词和体积限制。
  • 整理结果不对：先用识别测试验证文件名，再检查命名模板、分类规则、前置过滤和 AI 设置。
  • 通知收不到：检查 Telegram 白名单、企业微信回调、公网地址、静默时段和通知类型开关。
场景 优先查看 处理方向
首次部署 容器日志、系统设置、CD2 连接测试。 先确认 /config 可写、/CloudNAS 可见、端口未占用。
播放失败 代理配置、Emby ItemID、播放请求日志、115 账号状态。 从本地代理和回退本地开始验证，再排查 302 直链。
同步异常 STRM 任务日志、规则源路径、输出路径权限。 先手动全量同步一小段目录，确认规则正确后再扩大范围。
订阅漏资源 PT 站点搜索结果、RSS、质量规则、聚合来源。 降低过滤条件，确认站点能搜到，再恢复目标质量规则。
整理误判 识别测试、整理历史、词表、AI 调用记录。 先补词表和规则，再考虑开启更积极的 AI 模式。
