01 Overview

DIAN-115 是什么

DIAN-115 是围绕 115 网盘、CloudDrive2、Emby、PT、Telegram 和媒体库迁移的一套媒体自动化控制台。它不是单纯的文件管理器，也不是单纯的前端皮肤，而是把资源获取、旧库迁移、整理入库、播放分发和维护排障串起来。

资源进入

通过探索发现、PT 搜索、RSS、Telegram 频道监控、海盗湾、UIndex、TGx、TheRARBG、癫影资源和演员订阅，把可用资源汇集到项目里。

自动处理

把订阅命中的资源转存、离线下载或交给下载器，再通过整理规则识别、改名、洗版、多版本策略、分类配置和 STRM 规则进入媒体库。

播放与分发

通过 Emby / Navidrome 代理、媒体分享和 Telegram 通知，把媒体能力交付给自己或用户。

最重要的理解： 115 账号负责网盘能力，CD2 负责云盘挂载和路径映射，Emby / Navidrome 负责播放与媒体库，DIAN-115 负责把它们之间的自动化流程、监控和运营动作统一起来。

适合的场景

  • 希望从 PT、TG、公开 BT、癫影等来源发现资源，并统一订阅、下载和转存。
  • 希望把 115 或 CD2 挂载里的媒体按统一命名规则整理到电影、剧集和多版本目录。
  • 希望用 STRM 让 Emby 读取云盘媒体，减少本地占用，并支持入库通知和自动分享。
  • 已经用其它工具生成过 Emby 媒体库，希望把旧库迁移到本项目，并尽量保留媒体路径、画像和 FFP 识别成果。

不建议一上来就做的事

  • 没有配置 115 和 CD2 就直接创建 STRM 规则。路径映射不清楚时，后面排障会很痛苦。
  • 没有小范围测试整理规则就启用自动监控。先用识别测试和一小段目录验证命名、洗版和分类。
  • 没有配置通知和日志就跑大批量任务。出现限流、验证码、路径错误时很难第一时间发现。
02 Deploy & First Run

部署方法与第一次启动

先把服务稳定跑起来，确认网络模式、端口、目录挂载和数据持久化都正确，再进入旧库迁移和功能配置。首次配置不要急着开自动化。

推荐部署：Docker Compose

GitHub README 的快速部署使用 madbrolab/dian115 镜像和 host 网络模式，适合 Linux、NAS、Unraid 等服务端环境。新建工作目录后保存下面的 docker-compose.yml，再启动服务。

services:
  dian115:
    image: madbrolab/dian115
    container_name: dian115
    restart: unless-stopped
    network_mode: host
    volumes:
      - ./config:/config
      - ./media:/media
      - ./CloudNAS:/CloudNAS:shared
docker compose up -d
host 网络的含义： 使用 network_mode: host 时不用写 ports 映射，服务直接占用宿主机端口。访问地址通常是 http://服务器IP:8095。如果你的 Docker 环境不支持 host 网络，或需要通过反向代理统一入口，改用桥接网络写法。

桥接网络或反向代理场景

桥接网络需要明确映射端口。主服务默认 8095；Emby 代理默认 8098；Navidrome 代理默认 4534。代理端口后续可以在页面里改，Compose 里只暴露你实际会用到的端口。

services:
  dian115:
    image: madbrolab/dian115
    container_name: dian115
    restart: unless-stopped
    ports:
      - "8095:8095"
      - "8098:8098"
      - "4534:4534"
    volumes:
      - ./config:/config
      - ./media:/media
      - ./CloudNAS:/CloudNAS:shared
    environment:
      - PORT=8095
      - TZ=Asia/Shanghai
      # - DIAN115_HTTP_PROXY=http://代理服务器IP:7890

FlareSolverr 部署：给 Cloudflare 站点过盾

UIndex 等资源站可能会触发 Cloudflare 验证。需要这类站点时，单独部署 FlareSolverr，然后在 DIAN-115 的系统设置里填写服务地址并测试。没有使用这些站点时可以先不部署。

入口：/control/settings 的「基础设置」或代理相关配置中填写 FlareSolverr 服务地址。保存后回到探索发现，用 UIndex 等需要过盾的站点做一次小范围搜索测试。

services:
  flaresolverr:
    image: ghcr.io/flaresolverr/flaresolverr:latest
    container_name: flaresolverr
    restart: unless-stopped
    ports:
      - "8191:8191"
    environment:
      - LOG_LEVEL=info
      - LOG_FILE=none
      - LOG_HTML=false
      - CAPTCHA_SOLVER=none
      - TZ=Asia/Shanghai
    volumes:
      - ./flaresolverr:/config
DIAN-115 与 FlareSolverr 的位置 服务地址填写 说明
同一个 Compose 桥接网络 http://flaresolverr:8191 推荐把两个服务放进同一个 Compose 文件，容器名可直接解析。
DIAN-115 使用 host 网络，FlareSolverr 端口映射到宿主机 http://127.0.0.1:8191 或 http://服务器IP:8191 只有同一台宿主机上才适合用 127.0.0.1。
分别部署在不同机器 http://FlareSolverr机器IP:8191 确认防火墙和 Docker 端口都放行，尽量只允许内网访问。
最常见的填错： DIAN-115 在容器里运行时，127.0.0.1 指的是 DIAN-115 容器自己，不是宿主机，也不是另一个容器。除非使用 host 网络或同一网络命名空间，否则不要把另一个容器的服务地址写成 127.0.0.1。
暴露范围： FlareSolverr 会启动浏览器环境，首次请求可能比普通 HTTP 请求慢。8191 端口只建议内网访问，不要直接暴露到公网。

启动后检查

启动服务

执行 docker compose up -d 后先看容器日志，确认主服务没有反复重启。桥接网络下检查端口映射；host 网络下检查宿主机端口是否被其它服务占用。

打开管理后台

浏览器访问 http://服务器IP:8095。第一次进入会要求设置管理员密码；如果授权守卫提示需要激活，按页面提示填写 License Key，成功后再继续配置。

确认数据持久化

重启容器后密码、授权、115 账号、设置项仍应存在。若重启后全部丢失，先检查 /config 是否正确挂载，不要继续配置大量账号。

移动端或 APP 访问

项目支持 APP 场景下配置服务地址。APP 首次启动会进入服务器配置页，填入后端地址并测试连接；Web 端直接访问服务地址即可。

03 Migration First

旧工具 Emby 库迁移：Emby 媒体画像是第一步

如果你已经用其它工具生成过 Emby 媒体库，这一节要放在所有配置细节之前读。Emby 媒体画像不是普通辅助插件，而是把旧工具生成的媒体库迁移到 DIAN-115 的关键桥梁。

迁移目标： 先从旧 Emby 拉取媒体条目、文件路径和媒体画像，通过路径替换匹配 DIAN-115 新建的 115 目录树，再配合 Madby 插件和 FFP 能力，把旧库中的媒体识别成果接到本项目的 STRM、播放和资源画像体系里。

为什么要先做这件事

保护旧库成果

旧工具已经生成过媒体库、识别过媒体、积累过路径和画像。直接重建 STRM 可能导致旧库信息断层；先拉取画像，可以把旧路径关系转成 DIAN-115 可识别的数据。

避免覆盖旧 STRM

迁移初期 STRM 规则只应该构建目录树，不要马上生成 STRM。等旧 Emby 数据拉取、路径替换、画像匹配都确认后，再开启生成。

接入 Madby 生态

Madby 负责把 Emby 事件、播放、入库和 FFP 相关能力接入 DIAN-115。迁移完成后，Madby 与本项目 Webhook/FFP 接口配合，才能让新库持续保持可追踪。

迁移前必须确认

  • 旧 Emby 仍然可访问，并且你有旧 Emby 的 API Key。
  • 旧库中媒体文件路径能被转换到新环境中的 115 路径或 CD2 挂载路径。
  • 新 DIAN-115 服务已经可以启动，数据目录 /config 已持久化。
  • 已经备份旧 Emby 数据库、旧 STRM 输出目录、DIAN-115 的 /config/dian115.db。
  • 暂时不要让新 Emby 大规模扫描新 STRM 输出目录，等迁移链路确认后再扫描。

标准迁移顺序

先配置最小基础能力

只配置能支撑路径识别的必要项：管理员登录、115 账号、CD2 地址/API Token、CD2 挂载路径。其它订阅、PT、TG 等能力都可以后置。

创建 STRM 规则但先不要生成 STRM

入口：/automation/organizing 的「STRM 规则」。选择 115 源路径和输出路径，先执行全量同步或构建目录树。此时关闭或暂不使用「生成 STRM」，避免覆盖旧工具已经生成的 STRM。

打开 Emby 媒体画像插件拉取旧库

入口：/resources/plugins/emby-media-lite。填写旧 Emby 地址和旧 Emby API Key，配置路径替换规则，把旧 Emby 返回的媒体路径转换成 DIAN-115 目录树能匹配的 115 路径或 CD2 路径。

核对路径替换与匹配结果

路径替换是迁移成败的核心。旧路径、本地挂载路径、CD2 路径、115 路径必须能对应。匹配不到时不要继续生成 STRM，先修正替换规则或 CD2 挂载前缀。

配置 Madby 插件与 DIAN-115

在 Madby 侧填写 DIAN-115 地址和 OpenAPI Key；在 DIAN-115 的系统设置中选择 Madby 通知来源或保留原生 Webhook，避免两边重复推送。Madby 事件会进入本项目的事件处理、通知和 FFP 相关流程。

上报画像并接入 FFP

Emby 媒体画像插件可将未上报记录上报到癫影 SHA1 FFP。配合 Madby 和 emby-ffp-seed 类能力后，新库播放、入库、整理命中和 FFP 识别会逐步进入 DIAN-115 的统一统计与缓存。

最后再生成新 STRM 并让新 Emby 扫描

确认旧库画像已拉取、路径已匹配、Madby 已接入、FFP 上报策略清楚后，再回到 STRM 规则开启生成 STRM。建议使用全新 Emby 或干净媒体库扫描新 STRM 输出目录。

Madby 与 Emby 媒体画像怎么配合

组件 负责什么 迁移中怎么用 迁移后怎么用
Emby 媒体画像 从旧 Emby 拉取媒体路径、媒体条目和本地记录，并按路径替换匹配 DIAN-115 目录树。 用于把旧工具生成的库映射到新项目，确认旧路径与新 115/CD2 路径能对齐。 可继续上报未上报记录，清理本地画像记录，辅助 FFP 和媒体识别。
Madby 插件 把 Emby 入库、播放等事件按 DIAN-115 OpenAPI 接入本项目。 迁移阶段用于确认新旧 Emby 事件入口、OpenAPI Key 和通知来源不会冲突。 作为持续事件入口，驱动播放通知、入库通知、自动分享、防抖和发布树同步。
FFP / 癫影能力 保存或查询 SHA1、媒体画像、种子来源和识别命中。 迁移时承接旧库媒体文件的画像上报，减少新库重新识别成本。 用于整理命中、Madby 命中、资源统计和后续分享/识别闭环。
STRM 规则 构建 115 目录树并生成 Emby 可扫描的 STRM 文件。 先建树不生成，给媒体画像匹配路径；确认后再生成。 负责后续增量、定时同步、智能清理和新媒体库的持续更新。
迁移时最容易出错的点： 一边还没匹配旧库路径，一边已经打开新 STRM 生成和 Emby 扫描。这样会让旧库画像、路径映射、新 STRM、新 Emby 条目混在一起，后面很难判断到底是哪一层错了。
04 Baseline

基础配置顺序

下面是普通新装场景的配置顺序。若你在迁移其它工具生成的 Emby 库，先完成上一节的媒体画像迁移规划，再从这里补全配置。

1. 安全与 API

管理员密码与两步验证

入口：/control/settings 的「安全与API」。首次设置密码后，建议管理员开启两步验证。关闭两步验证需要密码和验证码确认。

OpenAPI Key 与 Webhook

OpenAPI Key 用于外部 API、Webhook 和部分插件调用。保存后可复制 Emby Webhook 推送地址；Webhook 可选择原生来源或 Madby 来源，并可自动同步发布树。

2. 115 账号

入口：/resources/accounts。这是项目最核心的基础能力，影响转存、分享、离线下载、秒传和回收站维护。

添加与激活

可以扫码获取 Cookie，也可以手动粘贴 Cookie。添加后设置活跃账号；多账号场景下开启自动切换并定期批量检查 Cookie。

扩展设置

保存 115 安全码、分享目录和离线下载目录。安全码用于清空回收站，分享目录是转存或分享的默认目标，离线下载目录用于磁链/ED2K 等离线任务。

限流与健康

根据账号情况设置 115 API QPS。遇到验证码时在账号页处理；分享页可续期、删除过期分享；回收站页可还原、删除或定期清理。

3. CloudDrive2

入口：/control/settings 的「基础设置」。填写 CD2 地址、API Token 和 CD2 挂载路径，保存后测试连接。

路径要先想清楚： 整理规则和 STRM 规则都会依赖 CD2 挂载路径。源路径、目标路径、Emby 看到的路径、115 网盘路径之间必须能互相对应。路径不一致会导致目录树建了，但 STRM 不能播放或整理找不到文件。

4. Emby / Navidrome

Emby 在「我的媒体」里的媒体库配置中配置 Emby 地址、API Key、代理端口、云盘名称、本地直连和兜底策略。配置后可查看仪表板、内容排行、缺集、用户和媒体分享。
Navidrome 如果需要音乐库代理，配置 Navidrome 服务、目录树同步、歌曲 ID 同步和符号链接生成。没有音乐需求可以后置。

5. PT 站点与下载器

入口：/resources/sites 与 /resources/downloads。先添加 PT 站点 Cookie、RSS、站点状态，再添加 qBittorrent 或 Transmission 下载器，测试连接后设为默认。随后在「媒体订阅」的 PT 设置里配置搜索、RSS、站点休眠、翻页、质量优先级和过滤规则。

6. Telegram

配置 入口 用途 注意
通知 Bot /resources/plugins/notifications 系统通知、任务状态、播放、媒体库、文件操作、账号状态。 填写 Bot Token 和白名单，可测试连接、发送测试、同步命令菜单。
TG 频道监控 /resources/plugins/tg-channel 连接 Telegram 客户端，监控频道资源，转存 115 链接或离线链接。 监控转发 Bot 是独立 Token，需要加入目标频道并设置白名单。

7. 代理、FlareSolverr、AI 与 FFP

  • HTTP 代理：系统设置里保存代理，或使用 DIAN115_HTTP_PROXY 环境变量。环境变量优先级更高。
  • FlareSolverr：用于 UIndex 等 Cloudflare 站点过盾。先按「部署方法与第一次启动」里的 Compose 启动，再在系统设置中填写服务地址并测试连接。
  • AI 模型：只支持 OpenAI 协议或 OpenAI 兼容代理，可配置多个提供商，并按聊天、Telegram、整理、字幕等使用位置分别选择模型。
  • FFP 工具：ffprobe / ffmpeg 会在运行时按需下载到配置目录；字幕、媒体画像、FFP 识别和部分封面能力会用到。
05 Workflows

核心使用流程

项目的核心价值在流程，不在单个页面。下面按真实使用路径说明。

完整闭环：从发现到播放

发现资源

在「探索发现」使用探索、PT、TG 频道、海盗湾、UIndex、TGx、TheRARBG 等标签页搜索。TG 频道资源来自监控插件沉淀的消息；PT 资源依赖站点配置；UIndex 等站点可能依赖 FlareSolverr。

订阅或直接转存

电影、电视剧、演员可进入订阅体系。订阅会按 PT、TG、癫影、聚合策略持续搜索；也可以直接把 115 分享转存到目标目录，或把磁链/ED2K 交给离线下载。

下载与落盘

PT 资源进入下载器；115 分享进入网盘目录；磁链进入 115 离线目录。到「下载管理」看 PT 下载任务和下载器状态，到「文件管理」看本地目录和预览文件。

整理入库

「媒体整理」按规则扫描源路径，识别电影/剧集，应用命名模板、洗版、多版本、字幕同步和分类配置，生成规范目录。先看整理历史，确认无误后再扩大范围。

生成 STRM

「STRM 规则」从 115 源路径构建目录树，输出本地 STRM 文件，支持 CD2 路径模式或链接模式。全量同步先建树，再生成 STRM；实时监控会根据 115 生活事件增量同步。

Emby 播放与分享

Emby 扫描 STRM 输出目录后，DIAN-115 通过代理、Webhook、Madby、媒体分享和通知把内容分发出去。媒体分享可发到 TG、上传到癫影，或在 115 链接风控时切换 ED2K 模式。

订阅流

电影与电视剧订阅

入口：/automation/subscriptions。可查看电影订阅、电视剧订阅、处理状态、进度和来源。PT 订阅可手动搜索，也可让 RSS 匹配新种子。

演员订阅

在演员订阅中搜索演员，设置媒体类型、年份、类型包含/排除等条件。扫描后会把演员作品沉淀为可跟踪项目，适合长期补全某个演员的作品。

追剧日历

入口：/automation/calendar。按月份查看订阅剧集更新计划，适合判断哪些剧需要等待更新，哪些可以手动催搜索。

聚合订阅

在订阅设置里可以控制订阅按钮默认行为，并维护全局质量优先级方案库。PT 和聚合可以使用相同或不同方案。

分享流

从我的媒体分享

入口：/resources/media 的「媒体分享」。选择媒体库、电影、剧集和具体版本后，可分享到 Telegram 或上传到癫影。

自动分享

开启自动分享后，收到入库通知会等待防抖窗口，按队列间隔创建分享。可选择媒体库范围、TG 转发目标、癫影分享账号和 ED2K 兜底。

风控处理

分享状态页会展示 115 分享限制和倒计时。风控中不要强行创建 115 明文分享，可使用 ED2K 模式继续分发。

06 Organize

媒体整理与 STRM 使用重点

这是项目最容易配置错、也最有价值的部分。先小范围验证，再启用自动监控。

整理规则怎么建

规则项 用途 建议
规则名称与启用 区分电影、剧集、动画、纪录片、临时测试等规则。 先建测试规则，确认无误后再启用正式规则。
云盘名称、源路径、目标路径 决定从哪里扫描、整理到哪里。CD2 路径会根据挂载前缀判断模式。 CD2 路径出现时必须选择云盘；源路径和目标路径不要互相包含。
整理方式 本地到本地可选择更丰富的转移方式；CD2 相关路径通常以移动为主。 跨文件系统、云盘挂载和权限不确定时先用小文件测试。
整理策略与重复文件策略 控制普通整理、评分洗版、多版本洗版、多版本共存等行为。 洗版前先配置评分维度和多版本槽位，避免误删或误覆盖。
前置过滤 按分辨率、编码、音频、来源等属性过滤，不满足条件的文件不进入整理。 过滤规则为空就是不过滤；开启“被排除时删源”要非常谨慎。
AI 增强识别 正则识别不到标题、TMDB 未命中或始终校正时调用 AI。 默认只在无标题和 TMDB 未命中时使用；始终调用会明显增加费用。
FFprobe、最小大小、排除格式 读取媒体属性、跳过小文件或特定后缀。 排除格式适合跳过 .ts 等不希望整理的文件。

分类配置与命名模板

  • 「分类配置」可以分别维护电影分类和电视剧分类，支持从项目内置模板导入、导出完整模板和编辑 JSON。
  • 「子分类与 Emby 同步」决定整理路径和 Emby 媒体库是否按同一套全局层级生成。
  • 「全局设置」维护电影命名模板、剧集命名模板、多版目录模板、整理并发、洗版评分规则和字段命名映射。
  • 命名模板使用 {变量} 占位符和 / 分隔目录层级，修改模板后建议只跑一小批历史文件验证。

整理历史怎么用

看结果

历史列表展示成功、失败、跳过、洗版、整理模式、识别方式、规则名称和详情。失败时先看错误信息，再去日志页搜同一任务。

修复错误

可以删除记录、重命名源文件、批量重命名、批量删除记录，也可以把失败记录提交给 AI 重试。删除历史记录本身不等于删除媒体文件。

STRM 规则怎么建

配置项 说明 常见选择
源路径 选择 115 网盘源路径。CD2 配置正常时会显示对应 CD2 路径。 选择已经整理好的 115 媒体目录。
输出路径 STRM 文件写入的本地目录，通常给 Emby 扫描。 为电影、剧集、动画分不同输出目录。
生成模式 CD2 路径模式会写入 CD2 路径；链接模式会写入项目直链地址。 有 CD2 时优先路径模式；无 CD2 或外部播放用链接模式。
同步模式 手动、实时监控、定时增量、定时全量。 先全量建树，确认后再开实时监控或定时同步。
文件后缀和排除目录 限定哪些文件生成 STRM，并排除不需要的 115 子目录。 只保留视频格式，排除花絮、样片、临时目录。
元数据同步与智能清理 同步海报、NFO、字幕等元数据，删除源文件时清理 STRM 和空目录。 媒体库稳定后开启智能清理。
迁移旧工具时： 先配置 115、CD2、癫影 OpenAPI、Emby FFP Seed、Madby 和媒体库路径，完整备份 Emby 数据。STRM 规则先只生成目录树，不要立刻打开生成 STRM，确认媒体画像迁移后再生成新 STRM。

08 Plugins

插件使用指南

插件按需启用。先理解插件解决什么问题，再配置账号和目标目录。

重点插件：Emby 媒体画像。 如果你是从其它工具迁移旧 Emby 库，请先看「旧工具 Emby 库迁移」章节。这个插件负责拉取旧库路径和媒体画像，并与 Madby、FFP、STRM 规则共同完成迁移闭环。
插件 什么时候用 首次配置 日常操作与注意
秒传上传 把本地文件或目录快速上传到 115，优先 SHA1 秒传，失败时可普通上传。 选择本地源、115 目标目录、成功/失败后处理、普通上传限速、并发和记录保留。 目录监控可自动处理新增文件；失败目录不要放在监控目录内部，避免循环处理。
Telegram 频道监控 监控 TG 频道里的 115 分享、磁链、ED2K 等资源，并转存或转发。 连接 TG 客户端，添加监控频道和转发目标，配置独立的监控转发 Bot Token 与白名单。 资源页可搜索、清空、转存到 115、离线下载；自动转发需要 Bot 在目标频道有权限。
通知插件 把系统事件发到 Telegram。 填写 Bot Token、白名单，选择通知类型、静默时段和公网通知图片地址。 保存后先测试连接，再发送测试消息；静默时段只拦截主动通知，命令回复不受影响。
Emby 媒体库封面 为 Emby 媒体库生成统一风格封面。 选择封面风格、媒体库、字体、字号、标题位置、分辨率、素材来源和自动刷新策略。 可 AI 翻译英文标题、批量生成、每天全量生成、入库后延迟刷新、清理素材缓存。
PT 刷流 按站点最新种子自动刷流，维护上传量和分享率。 选择刷流站点、下载器、保存路径、促销类型、H&R、大小、做种范围、删种规则。 超过上传/下载速度或活动任务上限会暂停新增；保留标签优先级最高，订阅命中的种子可排除。
癫影插件 使用癫影 OpenAPI Key 签到、上传分享资源、配置分享积分和 FFP 资源能力。 添加账号 Key，设置签到模式、签到时间段、默认分享账号和解锁积分。 FFP 统计可查看调用、命中、贡献和站点缓存；上传到癫影可用分享码或 ED2K 模式。
PT 站点统计 集中查看 PT 站点上传、下载、分享率、魔力、做种和健康状态。 先在 PT 站点配置中添加并启用站点。 可刷新全部或单站，按站点、域名、用户搜索，快速发现分享率和做种异常。
Emby 媒体画像 从其它工具生成的旧 Emby 库迁移到本项目时使用，是迁移链路的核心插件。 填写旧 Emby 地址和 API Key，配置路径替换，把旧库路径匹配到 DIAN-115 的 115/CD2 目录树。 先建目录树不生成 STRM；拉取画像后配合 Madby 与 FFP 上报，最后再生成新 STRM。
AI 字幕翻译 把已有非中文字幕翻译成中文外挂字幕。 配置字幕风格、AI 使用位置、最大字符数、媒体库白名单、外挂/内封处理策略。 只处理 MP4/MKV 且已有非中文字幕源；PGS/DVD 图像字幕不会 OCR；任务进度可看翻译对照和日志。
插件间依赖： Emby 媒体画像负责旧库迁移，Madby 负责 Emby 事件接入，癫影/FFP 负责画像与命中能力，通知插件负责消息，TG 频道监控负责资源采集和转发。不要把多个 Bot Token 混用。

日常维护清单

稳定运行靠巡检。账号、路径、队列、日志、缓存、备份都要有节奏地看。

频率 要做什么 入口 判断标准
每天 看仪表盘、任务队列、运行日志、115 Cookie 状态、下载器状态。 /overview、/control/jobs、/control/logs 无长时间卡住任务，无连续失败日志，Cookie 有效。

每次升级前 备份配置目录和数据库，记录当前版本、关键环境变量和端口。 宿主机文件系统 /config/dian115.db 和相关配置文件可恢复。

备份建议

  • 必须备份 /config/dian115.db，里面包含账号、设置、订阅、规则、用户数据和缓存索引等关键数据。
  • 如果上传了背景图、封面自定义素材、FFP 工具或插件生成文件，也应一起备份配置目录下对应子目录。
  • 升级前导出重要 PT 站点、下载器、整理规则、STRM 规则和订阅策略截图或 JSON，方便回滚核对。

日志与缓存

  • 日志页支持实时流，排查任务时先找任务 ID、规则名、路径、账号名、站点名。
  • 任务队列里的失败记录不要立刻清空，先打开详情看进度、错误和任务日志。
  • 缓存页的 TMDB、115 分享、TG 分享和离线链接缓存都可能影响搜索与分享结果。确认缓存过期或错误后再清理。

CD2 连接成功但整理找不到路径 CD2 挂载路径、云盘名称、源路径/目标路径是否一致 用文件管理和 CD2 本身确认宿主机看到的路径，再小范围重跑整理。
PT 搜索没有结果 站点 Cookie、站点状态、关键词、代理、FlareSolverr、站点休眠 先测试单站点，检查站点是否需要过盾；再看 PT 日志和站点返回。
下载任务不开始 下载器 WebUI 地址、账号密码、默认下载器、保存路径 在下载器配置里测试连接，确认 qBittorrent / Transmission 可从 DIAN-115 访问。
整理大量跳过 最小文件大小、排除格式、前置过滤、媒体类型识别、TMDB 打开整理历史看跳过原因，用识别测试单独跑一个文件名。
洗版结果不符合预期 洗版评分规则、多版本槽位、字段命名映射 先关闭删源选项，用少量样本调试评分维度和槽位匹配条件。
STRM 生成失败或播放失败 目录树是否构建、生成开关、输出路径权限、链接模式服务地址 先全量同步，再生成 STRM；确认 Emby 能读输出目录，链接模式服务地址外部可达。
Emby 入库通知没有触发 Webhook URL、OpenAPI Key、通知来源 native/Madby 用系统设置里的 Webhook 地址测试；确认 Emby 或 Madby 插件配置的是同一个后端地址。
Telegram 没收到通知 Bot Token、白名单、Chat ID、静默时段、网络 在通知插件里测试连接和发送测试；频道转发要看 TG 监控转发 Bot，而不是通知 Bot。
AI 字幕很慢或费用高 模型、最大字幕字符数、媒体库白名单、是否内封提取 使用便宜快速模型，缩小媒体库范围；默认只处理外挂字幕，必要时再开内封提取。

13 Appendix

附录：运行速查

这里补充日常维护时容易忘的运行细节，部署步骤以前面的「部署方法与第一次启动」为准。

服务入口

主入口默认 8095，可通过 http://服务器IP:8095 访问。反代时把管理后台和 API 都转发到这个端口，Emby / Navidrome 代理端口按各自配置单独处理。

数据位置与备份

数据库默认 /config/dian115.db。备份时优先备份整个 /config，再备份 STRM 输出目录和关键媒体路径映射说明。

常见环境变量

PORT 控制主服务端口，DEBUG=true 开启调试日志，LOG_LEVEL 控制日志级别，DIAN115_HTTP_PROXY 设置全局代理并优先于页面设置。

镜像更新

普通部署使用发布镜像即可。更新前先备份 /config，再执行 docker compose pull 和 docker compose up -d 重新创建容器。
