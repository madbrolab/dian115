# 前端仪表板内容分析

## 概述
115 STRM Manager 的前端仪表板是一个综合性的监控和管理界面，用于展示系统状态、统计数据和关键指标。

## 仪表板显示内容

### 1. 服务状态卡片（顶部）
显示三个核心服务的连接状态：

- **CloudDrive2 状态**
  - 图标：云图标 (ri-cloud-line)
  - 状态：已连接 ✓ / 未连接 ✗
  - 数据来源：`/api/status` 接口的 `cd2_connected` 字段

- **115网盘状态**
  - 图标：硬盘图标 (ri-hard-drive-2-line)
  - 状态：已登录（显示用户名）/ 未登录
  - 特殊提示：Cookie失效警告
  - 数据来源：`/api/status` 接口的 `login_115` 和 `user_115` 字段

- **Emby 状态**
  - 图标：电影图标 (ri-movie-line)
  - 状态：已连接 ✓ / 未连接 ✗
  - 数据来源：`/api/status` 接口的 `emby_connected` 字段

### 2. 入库统计卡片
展示媒体库的入库情况：

- **总入库数量**：右上角显示总数
- **最新入库电影列表**：
  - 显示最近入库的电影（默认20条，可配置）
  - 每个电影显示：
    - 海报图片（从Emby获取）
    - 电影名称
  - 数据来源：`/api/dashboard/stats` 接口的 `library` 字段
    - `total`: 总入库数
    - `latest_movies`: 最新电影列表（包含name, poster, added_at等）

### 3. 302跳转统计卡片
展示Emby代理播放的统计信息：

- **正在播放**：当前正在播放的数量
- **今日播放**：今天的播放次数
- **播放历史**：总播放次数
- **正在播放列表**：
  - 显示当前正在播放的内容
  - 每项包含：标题、用户名
  - 带有播放指示器动画
- 数据来源：`/api/dashboard/stats` 接口的 `redirect` 字段
  - `playing`: 正在播放数
  - `today`: 今日播放数
  - `total`: 总播放数
  - `playing_now`: 正在播放项目列表

### 4. 同步记录波形图
展示最近30天的同步活动：

- **图表类型**：柱状图
- **时间范围**：最近30天
- **显示内容**：每天的同步文件数量
- **交互**：鼠标悬停显示具体日期和数量
- 数据来源：`/api/dashboard/stats` 接口的 `syncChart` 字段
  - `labels`: 日期标签数组
  - `values`: 对应的同步数量数组

### 5. 关键指标卡片（4个指标）
展示系统的核心运营数据：

- **STRM文件总数**
  - 图标：文件图标 (ri-file-text-line)
  - 显示：系统中所有STRM文件的总数
  - ID: `metric-strm-total`

- **今日新增**
  - 图标：添加图标 (ri-add-circle-line)
  - 显示：今天新增的STRM文件数量
  - ID: `metric-strm-today`

- **24小时同步**
  - 图标：刷新图标 (ri-refresh-line)
  - 显示：最近24小时的同步次数
  - ID: `metric-sync-24h`

- **错误数量**
  - 图标：警告图标 (ri-error-warning-line)
  - 显示：系统错误的数量
  - ID: `metric-errors`

*注：这些指标的数据接口尚未在代码中完全实现*

### 6. 115网盘统计
展示115网盘的使用情况：

- **空间使用**：显示已用空间/总空间
- **账号总数**：配置的115账号数量
- **今日API调用**：今天对115 API的调用次数

*注：这些统计的具体数据接口尚未完全实现*

### 7. 工作队列统计
展示后台任务队列的状态：

- **运行中**：正在执行的任务数
- **等待中**：排队等待的任务数
- **已完成**：已完成的任务数
- 数据来源：`/api/workqueue/stats` 接口

### 8. 最近活动
展示系统的最近操作记录：

- 显示最近的系统活动日志
- 包括：同步操作、文件变更、错误信息等
- ID: `recent-activity`

*注：具体实现细节需要进一步查看日志系统*

## 数据刷新机制

### 自动刷新
- **状态检查**：每30秒刷新一次（服务状态卡片）
- **仪表板数据**：每60秒刷新一次（统计数据）

### 手动刷新
- 用户切换到仪表板页面时会触发数据加载

## API接口总结

### 主要接口
1. **`GET /api/status`** - 获取服务连接状态
2. **`GET /api/dashboard/stats?limit=20`** - 获取仪表板综合统计
   - 返回：library（入库统计）、redirect（302统计）、syncChart（同步图表）
3. **`GET /api/workqueue/stats`** - 获取工作队列统计

### 数据结构
```go
// 入库统计
type LibraryStats struct {
    Total        int           // 总入库数
    LatestMovies []LatestMovie // 最新电影列表
}

// 302跳转统计
type RedirectStats struct {
    Playing    int           // 正在播放数
    Today      int           // 今日播放数
    Total      int           // 总播放数
    PlayingNow []PlayingItem // 正在播放列表
}

// 同步图表数据
type SyncChartData struct {
    Labels []string // 日期标签
    Values []int    // 同步数量
}
```

## 特殊功能

### Emby封面图集成
- 系统会自动从Emby服务器获取电影封面图
- 通过 `enrichLibraryStatsWithPosters` 函数实现
- 需要配置Emby代理才能使用此功能

### 响应式设计
- 仪表板采用卡片式布局
- 使用CSS Grid实现响应式排列
- 支持移动端显示

## 待完善的功能

根据HTML结构，以下功能的后端接口可能尚未完全实现：

1. **关键指标卡片**的具体数据（STRM总数、今日新增等）
2. **115网盘统计**的详细数据
3. **最近活动**的具体内容和格式

## 总结

前端仪表板是一个功能完整的监控中心，主要关注：
- **服务健康状态**：CloudDrive2、115网盘、Emby的连接状态
- **内容统计**：入库数量、最新电影展示
- **播放监控**：302代理的播放统计和实时播放状态
- **同步活动**：30天同步趋势图表
- **系统运营**：工作队列状态、关键指标

整体设计以可视化和实时监控为核心，帮助用户快速了解系统运行状况。
